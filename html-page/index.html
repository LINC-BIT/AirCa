Overview

Project Background

Air cargo loading is a complex constrained optimization problem that impacts flight safety, operational efficiency, and fuel efficiency. However, existing benchmarking tools suffer from limitations such as small data volume, simplistic scenarios, insufficient constraints, and over-reliance on synthetic data, making them unable to meet industrial-grade practical requirements. Co-developed by the LINC-BIT team of Beijing Institute of Technology (BIT) and China TravelSky Holding Company, AirCa is built on real industrial production data and dedicated to evaluating air cargo loading optimization algorithms, filling the gap in industrial-grade benchmarking for this field.

Core Advantages

  • Industrial-scale: Based on 6.1 million cargo records and 1.1 million flight cycle real production data
  • Full scenario compatibility: Supports 10 types of mainstream civil aviation aircraft models, covering single-segment / multi-segment flight scenarios
  • Comprehensive constraint coverage: Fully supports 13 core constraints for cargo loading, closely aligning with real business modeling
  • Multi-algorithm adaptation: Built-in 11 reproducible combinatorial optimization baseline algorithms, including mathematical programming and heuristic search
  • Multi-objective evaluation: Multi-dimensional assessment of loading performance and computational overhead, balancing practicality and efficiency

Features

Industrial-grade Real Data Support

Built on real production system data from China TravelSky, covering 425 domestic and international airports and 10 types of mainstream civil aviation aircraft models. It includes massive cargo records and flight data, highly reproducing the real business scenarios and data distribution of air cargo loading.

Full-dimensional Constraint System

Fully implements 13 core constraints for air cargo loading across 3 major categories, covering the entire process of cargo handling, cargo hold management and loading operations, accurately matching the requirements of real business modeling and engineering implementation.

Multiple Benchmark Testing Tasks

Targeting the difficulties of air cargo loading combinatorial optimization, three typical testing tasks (multi-constraint, massive variables, multi-seg flights) are designed to fully cover algorithm performance evaluation under different complex scenarios.

Built-in Reproducible Baseline Algorithms

Integrates 11 classic combinatorial optimization algorithms, covering two major categories (mathematical programming and heuristic search), all with reproducible code to support direct invocation and horizontal comparison testing.

Multi-dimensional Evaluation Metrics

An evaluation system is built around loading business effectiveness and algorithm computational overhead, comprehensively measuring algorithm practicality and performance from multiple dimensions (center of gravity balance, transportation revenue, operational efficiency).

Flexible Extension & Customization

Supports custom test cases, quick integration of new algorithms, and weight configuration for multi-objective optimization. The modular architecture can flexibly adapt to different business scenarios and personalized research needs.

Benchmark Test Data

Achieve multi-objective optimization of center-of-gravity safety and transportation profit based on AirCa's 13 core constraint systems

Finite Feasible Solutions Core Feature

When there are m cargo items and n cargo holds, there are theoretically nm feasible loading schemes;
(Affected by various constraints, the number of valid feasible solutions is significantly reduced, actually only 0.1%-5% of the theoretical value)

Constraint Condition System Constraints

AirCa defines 13 core constraints aligned with actual operations, divided into three categories to ensure loading safety, feasibility and compliance:

Cargo Constraints
  • Weight Constraint: Single hold weight limit to avoid overload and imbalance
  • Volume Constraint: Single hold volume limit to prevent overfilling
  • CG Constraint: Defined center-of-gravity boundaries to avoid aircraft control risks
Cargo Hold Constraints
  • ULD Compatibility: Container type matches cargo hold specifications
  • Hold Exclusivity: Only one ULD per hold to prevent stacking
  • Cargo Type: Holds only accommodate specified cargo types
  • Front/Rear Holds: Short-haul cargo prioritized in rear holds to reduce costs
  • Special Constraints: Certain dangerous goods prohibited in specific holds
  • ULD Correspondence: Holds only accept compatible containers
Loading Operation Constraints
  • Dangerous Goods Isolation: Safe distance between incompatible dangerous goods
  • Combined Weight: Combined weight limit for adjacent cargo holds
  • Continuous Loading: No gaps between cargo to prevent shifting during transport
  • Loading Sequence: Short-haul cargo placed outside long-haul cargo for easy unloading

Multi-Objective Optimization Function Objectives

Optimization Objectives:
Minimize CG Deviation

Lower CG deviation values indicate better weight distribution, reducing trim drag and improving fuel efficiency.

Maximize Transportation Profit

Pricing based on industry-standard weight tiered rates (referencing TPE–DFW route)

Optimization Modes:
  • CG Priority (wcg=1.0, wrev=0.0): Strictly ensure flight safety
  • Profit Priority (wcg=0.0, wrev=1.0): Maximize revenue while meeting CG constraints
  • Balanced Mode: Slightly sacrifice CG precision to achieve significant profit improvement, realizing comprehensive optimization of safety, cost and revenue

Quick Start

Step 1: Download Demo Package

Download airca-benchmark-demo-cases.zip

Step 2: Prepare Environment

Open the terminal/command prompt, navigate to the unzipped directory, and install the required dependencies:


cd "<YOUR_UNZIPPED_PATH>/airca-benchmark-demo-cases"

pip install -r requirements.txt
replace <YOUR_UNZIPPED_PATH> with your actual unzip path (e.g. "C:\Users\YourName\Downloads").
Ensure you have Python 3.8+ installed before executing the above commands.

Step 3: Run Demo Cases

Run the following commands to execute different demo cases:


python ".\code\multi_constraint_cargo_loading\Aircraft_configuration_comparison.py" --root-dir "." --flight-number "3760461293" --algos all --time-limit 120

FAQ

The AirCa dataset covers 6 mainstream aircraft models, with core support for cargo loading research on A320 (narrow-body aircraft), B777 (wide-body aircraft), C919 and other models, while also including relevant cargo data for A319, A321, B787 and other models.

The downloaded dataset files need to be placed in the ./data/raw/ path of the project root directory. This directory must be organized into two subdirectories: split_by_aircraft_type (CSV files categorized by aircraft type) and split_by_date (CSV files categorized by date) to ensure scripts can read data correctly.

First, after cloning the project source code, execute pip install -r requirements.txt in the project root directory to automatically install all Python dependency libraries required for running cargo loading experiments, eliminating the need for manual installation of each library.

AirCa extracts three representative cargo loading workloads: 1. Multi-constraint cargo loading (modeling and optimization with intertwined multiple constraints); 2. Large-scale variable cargo loading (optimization of massive decision variables caused by fine-grained cargo splitting); 3. Multi-segment cargo loading (multi-stage and multi-objective cargo loading optimization), all of which are practical core issues in air cargo loading.

Precise downloading can be achieved through the DataDownloader API provided by AirCa. After instantiating AirCaDownloader, call the download_AirCa() method, specify the aircraft model via the aircraft_type parameter (e.g., A320) and the date via the date parameter. If both parameters are set to None, the full dataset will be downloaded.
